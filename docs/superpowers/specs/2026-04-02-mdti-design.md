# MDTI — MD로 보는 나의 개발 성향

> "네 CLAUDE.md가 너한테 하고 싶었던 말"

## 한 줄 요약

CLAUDE.md를 분석해서 개발 성향을 페르소나로 분류하고, 발칙하게 로스팅하고, 공유하게 만드는 소셜 바이럴 서비스.

## 타겟

Claude Code를 적극 활용하는 테크 프로페셔널. LinkedIn 공유를 통한 바이럴 확산.

---

## 1. 핵심 플로우

```
LinkedIn에서 친구 결과 카드 봄
  → mdti.dev 방문
  → CLAUDE.md 붙여넣기
  → 브라우저에서 분석 (서버 전송 X)
  → 결과: 페르소나 + 로스팅 + 강점 + 궁합 + MD 처방전
  → 결과만 서버 저장 → 공유 URL 생성
  → LinkedIn에 공유 (문구 자동 생성)
  → 다른 사람이 봄 → 반복
```

### 진입 경로

| 경로 | 설명 | 용도 |
|------|------|------|
| **웹 (메인)** | mdti.dev에서 MD 복붙 | 바이럴 유입, 마찰 제로 |
| **CLI (서브)** | `npx mdti` | 파워유저, 여러 MD 자동 수집 + 정밀 분석 |

### MD 가져오기 보조

웹 페이지에 원클릭 복사 안내:

```bash
pbcopy < ~/.claude/CLAUDE.md
```

"터미널에서 위 명령어를 실행하면 클립보드에 복사됩니다 → 여기에 붙여넣기"

## 2. 프라이버시 아키텍처

**원칙: 원본 CLAUDE.md는 서버를 절대 거치지 않는다.**

```
[사용자 브라우저]
  ├─ MD 붙여넣기
  ├─ JavaScript 분석 엔진 (클라이언트 사이드)
  ├─ 분석 결과 생성 (페르소나, 점수, 문구)
  └─ 결과만 API로 전송 → [서버: 결과 저장 + URL 생성]
```

- 분석 엔진: 규칙 기반 패턴 매칭 (JS로 브라우저에서 실행)
- 서버에 저장되는 것: 페르소나 유형, 점수, 생성된 문구, 통계용 메타데이터
- 서버에 저장되지 않는 것: 원본 MD 텍스트, 민감 정보
- 신뢰 문구 (페이지 상단): **"당신의 CLAUDE.md는 브라우저를 떠나지 않습니다"**

## 3. 분석 엔진 (규칙 기반 패턴 매칭)

### 분석 차원

클라이언트 JS에서 MD 텍스트를 파싱하여 다음 차원을 점수화:

| 차원 | 측정 방법 |
|------|-----------|
| **자동화 성향** | 훅, 스크립트, cron, 자동화 관련 키워드 빈도 |
| **제어 성향** | 규칙/금지/필수 키워드, 세부 지시문 밀도 |
| **도구 다양성** | 연동 도구 수 (Slack, Notion, API 등) |
| **MD 성숙도** | 전체 줄 수, 섹션 구조화, 메모리 활용 여부 |
| **협업 지향** | 팀 규칙, 코드 리뷰, PR 관련 키워드 |
| **보안 의식** | .env 규칙, 민감정보 언급, 권한 관련 규칙 |

### 페르소나 분류

점수 조합에 따라 페르소나를 배정. 한글 메인 + 영어 부제:

| 페르소나 | 특징 |
|----------|------|
| **봇 농장주** — The Puppet Master | 자동화 극고, 도구 다양성 극고. "AI를 부리는 줄 알았는데 AI가 시킨 대로 사는 사람" |
| **손이 빠른 무법자** — The Speedrunner | MD 최소, 규칙 거의 없음. "설정은 사치, 실행이 정의" |
| **보안 편집증 환자** — The Fortress | 보안 규칙 극고, 권한 통제 촘촘. "env 파일 꿈에서도 본다" |
| **CLAUDE.md 3줄러** — The Minimalist | MD 극히 짧음. "Claude야 알아서 해" 마인드 |
| **플러그인 수집가** — The Collector | 도구 설치 많지만 활용도 불명. "일단 깔고 본다" |
| **규칙 제왕** — The Legislator | 제어 성향 극고, 규칙이 법전 수준. "Claude도 헌법이 필요하다" |
| **조용한 장인** — The Craftsman | 균형 잡힌 설정, 과하지 않은 규칙. "도구는 수단일 뿐" |
| **과몰입러** — The Deep Diver | 특정 도메인에 극도로 깊은 설정. "한 우물만 파는데 그 우물이 지하 5층" |

> 페르소나 목록은 MVP 출시 후 실제 데이터 기반으로 조정 예정.

## 4. 결과 페이지 구성

### 4.1 🔥 불편한 진실 (로스팅)

MD에서 발견한 특징을 발칙하게 로스팅. 3개 내외.

- 분석 데이터에 기반한 구체적 지적 (예: "CLAUDE.md가 249줄인데 이력서는 2페이지도 안 되죠?")
- 톤: 찔리지만 웃긴, 공격이 아니라 애정 어린 디스

### 4.2 💎 근데 진짜 대단한 점 (인정)

로스팅만 하면 기분 나쁘니까. MD에서 읽히는 실질적 강점 3개.

- 단순 칭찬이 아닌 근거 기반 인정 (예: "비개발자인데 CLI 자동화를 실무에 녹여낸 건 진짜 드문 케이스")

### 4.3 💘 궁합

| 유형 | 설명 |
|------|------|
| **찰떡 궁합** | 보완 관계 유형 + 왜 잘 맞는지 유머러스하게 |
| **환장 궁합** | 충돌 관계 유형 + 왜 부딪히는지 |
| **거울 궁합** | 같은 유형끼리 만나면 생기는 일 |

궁합 비교 기능: 두 사용자의 결과 URL을 조합하면 1:1 궁합 페이지 생성.
`mdti.dev/match/{id1}/vs/{id2}`

### 4.4 🛠️ MD 업그레이드 처방전

MD에서 부족한 부분을 짚어주는 실질 가이드. 로스팅 톤 유지.

예시:
- "피드백 규칙이 하나도 없네요. Claude가 뭘 잘못해도 다음에 또 똑같이 합니다."
- "프로젝트별 CLAUDE.md가 없어요. 원룸에서 회사 생활하는 격."

### 4.5 📊 글로벌 통계

- 전체 사용자 대비 백분위 (예: "MD 복잡도 상위 4%")
- 평균 대비 수치 (예: "CLAUDE.md 길이 평균의 8.3배")
- 해당 페르소나 비율 (예: "봇 농장주는 전체의 3.2%")

### 4.6 🔗 LinkedIn 공유

- 공유 버튼 클릭 시 LinkedIn 포스트 문구 자동 생성
- OG 카드 최적화 (LinkedIn 카드 규격)
- CTA: "나도 해보기 → mdti.dev"

자동 생성 문구 예시:
> 내 CLAUDE.md를 분석해봤더니 '봇 농장주'가 나왔다 ㅋㅋ
> AI 도구를 어떻게 쓰는지가 곧 일하는 방식이라는데, 불편한 진실이 꽤 찔린다.
> 나도 해보기 → mdti.dev

## 6. 기술 스택

### 모노레포 구조

```
mdti/
├── apps/
│   └── web/                 # Next.js (프론트 + API Routes)
│       ├── app/
│       │   ├── page.tsx           # 랜딩 (MD 붙여넣기)
│       │   ├── r/[id]/page.tsx    # 결과 페이지
│       │   ├── match/[id1]/vs/[id2]/page.tsx  # 궁합 비교
│       │   ├── api/results/       # 결과 저장 API
│       │   └── api/og/            # OG 이미지 생성
│       └── lib/
│           └── analyzer/          # 분석 엔진 (클라이언트용 번들)
├── packages/
│   ├── cli/                 # npx mdti (CLI 도구)
│   ├── analyzer/            # 분석 엔진 코어 (웹/CLI 공유)
│   └── types/               # 공유 타입
├── turbo.json
└── package.json
```

### 스택

| 영역 | 기술 |
|------|------|
| 프레임워크 | Next.js (App Router) |
| DB | Supabase (PostgreSQL) |
| OG 이미지 | `@vercel/og` (Satori) |
| 배포 | Vercel |
| 모노레포 | Turborepo |
| CLI | Node.js (bin 패키지) |
| 스타일링 | Tailwind CSS |

### Supabase 테이블 (최소)

```sql
-- 분석 결과
results (
  id          text primary key,    -- 공유 URL ID (nanoid)
  persona     text not null,       -- 페르소나 키
  scores      jsonb not null,      -- 차원별 점수
  roasts      jsonb not null,      -- 생성된 로스팅 문구
  strengths   jsonb not null,      -- 강점 문구
  prescriptions jsonb not null,    -- MD 처방전
  stats       jsonb,               -- 통계 스냅샷 (생성 시점)
  created_at  timestamptz default now()
)

-- 글로벌 통계 (집계용)
persona_stats (
  persona     text primary key,
  count       int default 0,
  avg_scores  jsonb
)
```

## 7. 분석 엔진 상세

### 패턴 매칭 규칙 (예시)

```javascript
// 자동화 성향
const automationPatterns = [
  /hook[s]?/gi,           // 훅 사용
  /cron|schedule/gi,      // 스케줄링
  /자동|auto/gi,          // 자동화 키워드
  /script|스크립트/gi,    // 스크립트
  /deploy|배포/gi,        // 자동 배포
];

// 제어 성향
const controlPatterns = [
  /금지|하지\s*마|절대/gi,   // 금지 규칙
  /반드시|필수|항상/gi,      // 필수 규칙
  /확인.*후|승인/gi,         // 승인 요구
];
```

### 점수 → 페르소나 매핑

각 차원 0-100 점수화 후, 가장 높은 점수 조합으로 페르소나 배정.
동점 시 MD 길이 / 구조화 정도로 결정.

### 로스팅 문구 생성

페르소나별 템플릿 + MD에서 추출한 구체적 데이터를 조합:
```
템플릿: "CLAUDE.md가 {lines}줄인데 {observation}"
데이터: lines=249, observation="본인 이력서는 아마 2페이지도 안 됨"
결과: "CLAUDE.md가 249줄인데 본인 이력서는 아마 2페이지도 안 되죠?"
```

## 9. 바이럴 루프 설계

```
[LinkedIn 피드에서 결과 카드 발견]
  → 호기심 클릭 → mdti.dev/r/{id}
  → "나도 해보기" CTA
  → MD 붙여넣기 (5초)
  → 결과 확인 → "ㅋㅋ 맞는데..."
  → LinkedIn 공유 버튼 (문구 자동 생성)
  → 내 피드에 올림
  → 내 연결들이 봄 → 반복

[궁합 루프]
  → 결과 보고 "너는 뭐 나왔어?" 
  → 친구에게 링크 공유
  → 친구도 분석
  → 궁합 비교 URL 생성
  → 둘 다 공유 → 바이럴 2배
```

핵심 지표:
- 결과 페이지 → "나도 해보기" 전환율
- 분석 완료 → LinkedIn 공유 전환율
- 공유 → 신규 유입 전환율
