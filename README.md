# .mdTI

> .md가 당신에게 하고 싶었던 말

**Live**: https://mdti.vercel.app

---

## 왜 만들었나요?

Claude Code 사용자마다 CLAUDE.md 작성 스타일이 다릅니다. 보안 규칙만 빼곡한 사람, 자동화 스크립트를 잔뜩 건 사람, 3줄만 쓴 사람. 이 차이가 AI 활용 성향을 그대로 보여준다는 데서 출발했습니다.

**mdTI는 CLAUDE.md를 분석해서 AI 활용 성향을 12가지 페르소나로 분류하고, md 품질을 점수로 매기는 소셜 바이럴 서비스입니다.** MBTI처럼 결과를 공유하고, 처방전으로 CLAUDE.md를 개선하면 점수가 올라가는 루프를 만듭니다.

## 어떻게 쓰나요?

터미널에서 수집 스크립트를 실행하면 CLAUDE.md, settings.json, MCP 설정, 커스텀 명령어, Hook, 플러그인 정보를 한 번에 수집합니다. 민감 정보(API 키, 토큰, 이메일)는 자동 마스킹됩니다. Mac / Windows 지원.

## 12가지 페르소나

| 페르소나 | 태그라인 |
|---------|---------|
| 🤠 로데오 마스터 | 하네스를 완전히 길들인 카우보이 |
| 👶 하기스 아키텍트 | 하네스 위에 올라타긴 했는데... 아직 기저귀 단계 |
| 🎪 봇 농장주 | AI를 부리는 줄 알았는데, AI가 시킨 대로 사는 사람 |
| ⚡ 손이 빠른 무법자 | 설정은 사치, 실행이 정의 |
| 🏰 보안 편집증 환자 | .env 파일이 꿈에 나오는 사람 |
| 📄 CLAUDE.md 3줄러 | Claude야 알아서 해 |
| 🧲 플러그인 수집가 | 일단 깔고 본다 |
| ⚖️ 규칙 제왕 | Claude에게도 헌법이 필요하다 |
| 🔧 조용한 장인 | 도구는 수단일 뿐 |
| 🕳️ 과몰입러 | 한 우물만 파는데 그 우물이 지하 5층 |
| 📢 협업 전도사 | 혼자 잘 하는 것보다 같이 잘 하는 게 중요하다며? |
| 🎲 위험물 취급자 | 보안? 그거 나중에 해도 되는 거 아닌가요? |

주 페르소나 + 부 페르소나 조합으로 12 × 11 = 132가지 프로필.

## .md력 측정기

5개 품질 차원으로 CLAUDE.md의 실제 품질을 측정합니다.

| 차원 | 측정 내용 |
|------|----------|
| 실행 가능성 | 빌드/테스트 커맨드, 아키텍처 경로, 검증 루프 |
| 간결성 | 역U자 길이 커브, 노이즈 감점, @import 분리 보너스 |
| 구조화 | 섹션 헤딩, 리스트 비율, 우선순위 마킹, 계층 구조 |
| 맥락 독점성 | 프로젝트 고유 금지 규칙, 워크플로우, 도구 맥락 |
| 방어력 | 민감 정보 보호, 검증 의무화, deny 규칙 |

5개 차원 합산(0~500) × 2 = 0~1000점, 6단계 티어:

| 티어 | .md력 | 느낌 |
|---|---|---|
| 🌋 Sequoia | 800+ | 하네스를 길들인 자 |
| 🏔️ Oak | 600~799 | 팀의 .md 기준점 |
| 🌳 Tree | 400~599 | 주변에서 물어보는 수준 |
| 🌿 Sapling | 250~399 | 기본기 장착 |
| 🌱 Sprout | 100~249 | 시작은 했다 |
| 🥚 Egg | 0~99 | .md가 뭐예요? |

## 분석 깊이

**성향 분석 (mdTI)**: 7개 차원 × 94개 패턴 — 자동화(13), 제어(10), 도구 다양성(20), 컨텍스트 관리(12), 팀 임팩트(14), 보안(10). 고유 신호 카운팅(패턴당 0/1)으로 키워드 반복이 아닌 다양성을 측정. CI/CD, Terraform 등 DevOps 키워드부터 Miro, Airtable 등 비개발자 도구까지 ���버.

**품질 분석 (md력)**: 5개 차원 — 실행 가능성, 간결성, 구조화, 맥락 독점성, 방어력. Anthropic 공식 가이드 + HumanLayer 연구 기반 설계.

**구조화 신호**: permissions deny 규칙, Hook 유형(AI 판단 vs 셸 실행), 플러그인 활성/설치 비율, 프로젝트별 CLAUDE.md 수.

## 주요 기능

- **팩폭 로스팅** — "Claude 장애 공지 뜨면 심장 먼저 멈추는 사람"
- **강점 인정** — 로스팅만 하면 서운하니까
- **궁합** — 찰떡 / 환장 / 거울 궁합
- **MD 업그레이드 처방전** — 페르소나 시그니처 + 품질 약점 기반 5개 고정 맞춤 처방 (tag 기반 중복 제거)
- **심층 분석 시각화** — 감지된 플러그인, MCP, Hook, 보안 신호 표시
- **.md력 리더보드** — GitHub 로그인 + 티어 + 순위 + Δ 변화량
- **캡쳐 기능** — 결과 카드 이미지 저장/복사 (소셜 공유용)
- **프로필** — 이름, 직무, 소속, 상태 메시지, LinkedIn
- **Contact** — 피드백/버그 제보/기능 제안 폼 (Supabase 저장)

## 프라이버시

**분석은 100% 브라우저에서. 원본은 어디에도 전송되지 않습니다.**

- 분석 엔진은 클라이언트 사이드 JavaScript (규칙 기반 패턴 매칭)
- 서버에 저장되는 것: 분석 결과 (페르소나, 점수, 문구)만
- 서버에 저장되지 않는 것: 원본 텍스트
- 수집 스크립트가 API 키/토큰/이메일을 자동 마스킹

## 기술 스택

- **프레임워크**: Next.js 16 (App Router, Turbopack)
- **스타일링**: Tailwind CSS 4
- **DB**: Supabase (PostgreSQL)
- **인증**: GitHub OAuth (Supabase Auth)
- **OG 이미지**: @vercel/og (Satori)
- **캡쳐**: html2canvas
- **배포**: Vercel
- **테스트**: Vitest (144개 테스트, 9개 파일)
