# .mdTI

> .md가 당신에게 하고 싶었던 말

**Live**: https://mdti.vercel.app

---

## 왜 만들었나요?

AI 코딩 에이전트 사용자마다 .md 파일 작성 스타일이 다릅니다. 규칙을 직접 만드는 사람, 남이 만든 걸 가져다 쓰는 사람, 맥락을 장황하게 설명하는 사람, 핵심만 딱 쓰는 사람. 이 차이가 AI 활용 습관을 그대로 보여준다는 데서 출발했습니다.

**mdTI는 CLAUDE.md, AGENTS.md, Cursor Rules 등 AI 에이전트 설정 .md 파일을 분석해서 "너도 모르게 반복하는 AI 사용 패턴"을 16가지 유형으로 분류하는 소셜 바이럴 서비스입니다.**

## 어떻게 쓰나요?

1. [mdti.vercel.app](https://mdti.vercel.app) 접속
2. .md 파일 내용을 붙여넣기
3. "내 MD 털어보기" 클릭
4. 결과 확인 + 궁합 비교 + 공유

분석은 **100% 브라우저에서** 실행됩니다. 원본 텍스트는 서버로 전송되지 않습니다.

## 16가지 유형

4개 축의 조합으로 16가지 유형을 분류합니다.

### 4개 축

| 축 | A 방향 | B 방향 | 보는 것 |
|----|--------|--------|---------|
| 도구 확장 | 탐색형 (G) — 남이 만든 걸 찾아 쓴다 | 구축형 (H) — 직접 만든다 | 플러그인/MCP vs 커스텀 스킬/에이전트 |
| AI 통제 | 통제형 (R) — AI에게 빡빡하다 | 위임형 (D) — AI를 믿고 맡긴다 | deny 규칙, MUST/NEVER, Hook 등 |
| 표현 방식 | 맥락형 (V) — "왜"를 설명한다 | 핵심형 (C) — "뭘"만 말한다 | 문서 길이, 설명 비율 |
| 구조 선호 | 구조화형 (S) — 체계적으로 정리한다 | 자유형 (F) — 자유롭게 쓴다 | 섹션, 헤딩, 리스트 비율 |

### 16 유형

| 코드 | 유형 | 한마디 |
|------|------|--------|
| GRVS | 📚 아키비스트 | 모든 MCP를 분류해서 정리해둔 사람 |
| GRVF | ⚡ 스톰 체이서 | 배경 설명은 길게 하는데 형식은 신경 안 쓰는 사람 |
| GRCS | ⚔️ 스파르탄 | 좋다는 건 깔아봤지만 남긴 건 핵심만인 사람 |
| GRCF | 🏹 프리 레인저 | 형식은 자유로운데 금지 사항은 확실한 사람 |
| GDVS | 🌐 월드 빌더 | 배경을 충분히 설명하면 AI를 믿고 맡기는 사람 |
| GDVF | 💫 의식의 흐름러 | 왜를 설명하면 나머지는 AI가 알아서 하는 사람 |
| GDCS | 🛸 오토파일럿 | 핵심만 말하고 나머지는 시스템에 맡기는 사람 |
| GDCF | 🏄 웨이브 라이더 | 필요한 것만 던지고 결과를 보는 사람 |
| HRVS | 🏰 결계사 | 규칙을 만들기 위한 규칙을 먼저 만드는 사람 |
| HRVF | 📖 규칙의 시인 | 설명은 시처럼 아름답고 규칙은 철저한 사람 |
| HRCS | ⚙️ 시계공 | 한 글자도 낭비 없는 완벽한 시스템을 만드는 사람 |
| HRCF | 📦 블랙박스 | 안에서 뭐가 돌아가는지 남이 모를 정도로 잘 만든 사람 |
| HDVS | 🌳 테라포머 | 토대를 완벽하게 깔고 AI는 자유롭게 움직이게 하는 사람 |
| HDVF | 🎭 마에스트로 | 직접 만든 판 위에서 AI가 자유롭게 연주하게 하는 사람 |
| HDCS | 🧘 젠 마스터 | 뒤에서 완벽하게 준비하고 앞에선 아무것도 안 하는 사람 |
| HDCF | 🌪️ 고스트 엔지니어 | 본인이 없으면 아무도 못 쓰는 시스템을 만든 사람 |

## 결과지 구성

1. **찔리는 한마디** — 가장 먼저, 가장 크게
2. **유형 이름 + 이모지** — 내 AI 사용 유형
3. **행동 묘사** — "너는 이런 사람이다"를 일상 언어로
4. **위트** — "혹시 ~한 적 없나요?"
5. **탐험 제안** — 반대 방향으로의 성장 제안
6. **페르소나 궁합** — 최적 궁합 / 보완 관계 / 최악 궁합 (스토리 포함)
7. **공유** — LinkedIn, X, 이미지 저장, 링크 복사

## 궁합

축 기반으로 자동 계산됩니다.

- **최적 궁합** (💙) — 3축 동일, 1축만 다른 유형. 거의 같은데 한 가지가 달라서 보완.
- **보완 관계** (💛) — 2축 동일, 2축 다른 유형. 서로 부족한 걸 채우는 관계.
- **최악 궁합** (💥) — 4축 전부 반대. 매일이 전쟁.

궁합 비교 페이지(`/match/{id1}/vs/{id2}`)에서 두 사람의 결과를 나란히 비교할 수 있습니다.

## 프라이버시

**분석은 100% 브라우저에서. 원본은 어디에도 전송되지 않아요.**

- 분석 엔진은 클라이언트 사이드 JavaScript (규칙 기반 패턴 매칭)
- 서버에 저장되는 것: 분석 결과 (유형, 점수, 문구)만
- 서버에 저장되지 않는 것: 원본 텍스트
- 수집 스크립트가 API 키/토큰/이메일을 자동 마스킹

## API

다른 프로덕트나 AI 에이전트가 .md 분석을 가져다 쓸 수 있습니다.

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/api/analyze` | API 사용법 자기 문서화 (에이전트가 읽고 스스로 호출 가능) |
| `POST` | `/api/analyze` | .md 텍스트 분석 → 유형 + 점수 + 처방전 + 공유 URL |

`text/plain` 또는 `application/json` (`{"text": "..."}`) 두 가지 형식 지원.

## 기술 스택

- **프레임워크**: Next.js 16 (App Router, Turbopack)
- **스타일링**: Tailwind CSS 4
- **DB**: Supabase (PostgreSQL)
- **OG 이미지**: @vercel/og (Satori)
- **캡쳐**: html2canvas
- **배포**: Vercel
- **테스트**: Vitest
- **분석 엔진**: `@mdti/analyzer` (독립 npm 패키지, 프레임워크 의존성 0)
